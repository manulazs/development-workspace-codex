# Evolution Workspace

This directory stores generated and curated artifacts for governed continuous evolution.

- `task-catalog.md`: current human-readable evolution task catalog generated by `scripts/evolve-workspace.py --write-catalog`.
- `task-catalog.json`: machine-readable version of the same catalog.
- `reports/`: dated continuous-evolution reports generated by `scripts/evolve-workspace.py --write-report`.
- `proposals/`: non-runtime-loadable skill or agent proposals generated by `scripts/scaffold-capability.py --mode proposal`.
- `templates/`: public templates for private observation logs, cross-cutting principles, observation review reports, and staged update notes.

Regenerate the catalog and write a report after adding, removing, reclassifying, or materially changing skills, agents, profiles, provenance, or governance docs.

## Private Observation Storage

Real task observations belong in the consumer workspace, not in this public template:

```text
.codex-local/evolution/observations/log.md
.codex-local/evolution/observations/cross-cutting-principles.md
.codex-local/evolution/staged-updates/YYYY-MM-DD/
```

Those paths are ignored by Git. Keep only sanitized templates, reports, proposals, and structural decisions in this repository.

Files under `docs/evolution/templates/` must stay as sanitized placeholders. Do not add raw prompts, local usernames, absolute private paths, runtime logs, client identifiers, credentials, or real observation payloads to public templates.

Observation status values are:

- `OPEN`: not reviewed or not actioned yet.
- `ACTIONED`: applied to a skill, agent, runbook, policy, docs, or script.
- `DECLINED`: reviewed and intentionally rejected.
- `SUPERSEDED`: replaced by a newer observation, decision, or implementation.

Cross-cutting principles require at least two linked observations or one explicit ADR reference before they become durable guidance.
